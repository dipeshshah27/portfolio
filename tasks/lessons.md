# Lessons

- **2026-06-11 — Replacing a `public/` image in place serves stale content.** The `next/image` optimizer caches by URL in `.next/cache/images`, so a new file under the same name keeps serving the old variant — browser hard-refresh can't fix it. When swapping an asset, rename it (e.g. `profile.jpg` → `profile-cafe.jpg`); same applies to CDN caching in production. Don't blame the browser cache first.
- **2026-06-11 — Opacity is the wrong tool for "receding" stacked cards.** Dimming a card in a sticky deck with `opacity` makes it translucent and everything behind bleeds through; use `filter: brightness()` instead, and never dim the top card (nothing covers it).
- **2026-06-11 — `color-mix()` interpolates alpha too.** Mixing an opaque color with a translucent one (`rgba(...,0.08)`) yields a translucent result — a "solid" surface built that way isn't solid in dark mode. Force an opaque `background-color` base and overlay tints as gradient layers.
- **2026-06-11 — "Story" was figurative, not literal.** When a brief says the site should "look like a story," that means narrative *flow and motion design* (scroll choreography, pacing, payoff), not literal chapter labels and storybook copy. Don't dress sections as "Chapter 01"; let scroll-driven animation do the storytelling.
- **2026-06-11 — "Go crazy with animations" means scroll-scrubbed, not fade-ins.** One-shot `whileInView` reveals read as tame. Premium feel = pinned sections scrubbed by scroll progress, sticky stacking cards, kinetic/mask typography, velocity-reactive elements, preloader, custom cursor. Research references (Awwwards/Muzli patterns) before designing, not after feedback.
- **2026-06-11 — Translucent glass cards can't stack.** Sticky stacking decks need a near-opaque surface (`.glass-solid`), or the card below bleeds through the card on top.
- **2026-06-11 — `background-clip: text` breaks across transformed children.** Gradient text + word-mask reveals: apply the gradient class per word span, not on the parent.
- **2026-06-10 — Headless Chrome `--virtual-time-budget` doesn't advance Motion's frame loop.** Verify Motion animations with Playwright driving real Chrome (`channel: "chrome"`) and wall-clock waits.
